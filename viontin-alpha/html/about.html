About Viontin

Our Philosophy

Zero to One, Scale-up Easily. Viontin grows with your project, from prototype to production fleet — one framework, any scale.

No DSL

Pure Rust — functions, structs, and traits. No custom language, no code generation.

Additive

Each level adds guardrails. Level 5 can do everything Level 0 can.

Detection

Framework infers scale from directory structure. No config files to maintain.

Escape Hatches

Every constraint can be overridden. Sensible defaults without lock-in.

Technical Decisions

Why std::net instead of tokio?

std::net::TcpListener and thread-per-connection. Zero async runtime dependency. Add tokio at the application level if needed.

Why no proc macros?

Declarative macros avoid compile-time overhead. No reflection, no code generation.

Why Laravel-inspired patterns?

Service providers, facades, event dispatchers — proven patterns adapted to Rust's type system.
